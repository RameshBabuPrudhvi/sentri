Sentri's 8-stage AI pipeline takes a URL all the way to executable, approved Playwright tests. Each stage is reviewed below with issues observed in the source.

#StageModule(s)Issues found
1Crawl / State Explorecrawler.js, stateExplorer.jsState fingerprint uses URL+title only — SPA route changes with same title are not distinguished. No robots.txt / sitemap awareness. Exploration doesn't retry on transient navigation errors.

StageModule(s)Issues found
Review queueroutes/tests.jsBulk approve/reject has no audit trail — who approved what is not stored per-decision, only in activity log at batch level.

Critical — Production Blockers

In-memory JSON database — no real persistence
db.js is a single JSON file flushed every 30 seconds. Any server crash, OOM kill, or Docker restart mid-test-run loses all un-flushed data. The README openly flags this but it is a blocking issue for any team running overnight regression suites. No transactions, no indexing, O(n) scans on every query.
Data Loss RiskReplace with SQLite/Postgres + Prisma
Generated Playwright code executes in an uncontrolled VM context
codeExecutor.js wraps LLM output in a function and runs it with Node's vm module or equivalent, sharing the same Node.js process. A prompt-injection payload in a tested website's page title could cause the LLM to emit malicious code that then executes with full server privileges. This needs a subprocess sandbox (worker_threads or a child_process with --experimental-vm-modules).
SecurityPrompt Injection Vector
JWT stored in localStorage — XSS-stealable
The README production checklist explicitly marks "Move JWT from localStorage to HttpOnly cookies" as incomplete. Any XSS vulnerability (including inside tested third-party pages rendered in the live browser view) can exfiltrate the token.
Security

High — Functionality Gaps

No test scheduling / CI webhook trigger
Tests can only be triggered manually from the UI. There is no cron scheduler, no REST webhook for CI/CD pipelines, and no GitHub Actions integration. Teams cannot run nightly regression without keeping a browser tab open.
Missing Core Feature
AI feedback loop bypasses human review for regenerated tests
In feedbackLoop.js, regenerated tests are auto-set to Approved status. This breaks the stated principle that "nothing executes until a human approves it" and creates a trust gap — regressions could silently introduce flawed tests.
Trust / Safety
Shadow DOM and web components not supported
elementFilter.js has no logic to pierce shadow roots. Modern SPAs (Lit, Stencil, Angular Elements, Salesforce LWC) are largely invisible to Sentri's crawler and element filter, producing empty or thin test suites.
Coverage Gap
No multi-tenancy — single shared database
All projects and users share one flat namespace. There is no organisation, workspace, or team scoping. A user can enumerate all tests across all projects via GET /api/tests with no projectId filter enforced at the auth layer.
SecurityMissing for SaaS

Medium — Quality & UX

No Playwright syntax validation before saving tests
testValidator.js checks for URL presence and step count but not AST-level Playwright correctness. A test with a syntax error will pass the Draft→Approved flow and only fail when executed, wasting browser compute and user time.
Quality
GraphQL API tests are collapsed into one endpoint
harCapture.js deduplicates by URL+method. All GraphQL operations (queries, mutations) share POST /graphql, so they are all merged into a single test. Sentri has no awareness of GraphQL operation names.
API Coverage
Rate limit retry has no maximum back-off cap
journeyGenerator.js retries on rate limit errors with isRateLimitError but there is no max retries count or cap on exponential back-off delay. A sustained provider outage can leave a run hanging indefinitely.
Reliability
No Slack/email/webhook failure notifications
When a scheduled or triggered test run completes with failures, there is no outbound notification. Teams have to poll the dashboard. The README lists this as incomplete.
Alerting

Capabilities needed to compete with enterprise autonomous QA platforms like Mabl, Testim, Katalon, Reflect, and QA Wolf.

Tier 1 — Must-have to compete

⏱
Scheduled Runs & CI Webhooks
Cron-based auto-runs (daily/nightly), plus a POST /api/projects/:id/trigger endpoint with a secret token. Emit results back to GitHub Actions, GitLab CI, CircleCI via status callbacks. Every major competitor has this.
🏢
Organisations & Teams
Workspace-level scoping of projects, tests, and users. Role-based access: Owner, Admin, Editor, Viewer. This is table-stakes for any B2B SaaS. Needed before any enterprise sale.
🔔
Failure Alerts (Slack/Email/PagerDuty)
Configurable webhooks per-project: on failure, on flake detection, on pass-rate drop below threshold. First-class Slack app with /sentri run and /sentri status slash commands.
🛡
Real PostgreSQL + Job Queue
Replace in-memory JSON with PostgreSQL + Prisma. Add BullMQ + Redis for crawl/run jobs so the API returns immediately and workers process async. Enables horizontal scaling and crash-safe execution.
🕸
Shadow DOM & Web Component Support
Pierce shadow roots during crawl. Detect custom elements and emit Playwright locators that use pierce: shadow selectors. Critical for Angular, LWC, Lit-based enterprise apps.
🔁
Regenerated Tests Back to Draft
AI feedback loop must route regenerated tests through human review like first-generation tests. Add a "re-review" queue tab and notify the approver on Slack/email.

Tier 2 — Competitive differentiators

🎭
Visual Regression Testing
Per-step screenshot diffing using pixelmatch or Playwright's toMatchSnapshot. Auto-generate baseline on first approved run. Flag pixel-level regressions alongside functional failures. Compete directly with Percy/Applitools integrations.
🗂
Test Suites & Tags
Group tests into named suites (Smoke, Regression, Release). Run a specific suite via CI. Tags already exist on the data model — surface them as first-class run filters in the UI and API.
📊
Coverage Heatmap
Overlay a site-graph node with a colour-coded "test density" score: pages with no tests are red, pages with multiple approved tests are green. Helps teams see coverage gaps visually.
🤝
Jira / Linear Issue Sync
Auto-create a bug ticket when a test fails (with screenshot, error, and Playwright trace attached). Sync test status back to linked issue. The traceability data model is already there — just needs the outbound webhook.
🧩
GraphQL-Aware API Testing
Parse operation name from POST body during HAR capture. Generate separate tests per GraphQL operation. Support introspection-based schema-aware test generation when an introspection endpoint is available.
🔐
Multi-Auth Profile Support
Store multiple named authentication profiles per project (admin, user, guest). Run the same test suite under each profile and automatically diff results. Surface RBAC test generation as a first-class dial option.

Tier 3 — Platform moat features

🤖
Autonomous Monitoring Mode
Run approved smoke tests every N minutes against production. On failure, auto-file a Slack alert and trigger a re-run to distinguish flake from real breakage. Position as "always-on QA agent" — the core brand promise made real.
💬
Conversational Test Editor
The /chat route exists but is disconnected from tests. Wire it up: "make this test also verify the error message" → AI diffs the existing code → shows a code diff → one click to apply. Natural-language test editing.
📦
Playwright Project Export
Export all approved tests as a standalone Playwright project (playwright.config.ts + test files) that runs with npx playwright test — zero Sentri dependency. This removes vendor lock-in anxiety, which is the top enterprise objection to AI QA tools.
🔮
Flakiness Prediction
Before running, score each test's flakiness risk based on: selector type (text selectors are high-risk), wait strategy, assertion type, historical heal rate. Show a risk badge in the UI. Flag high-risk tests for pre-emptive strengthening.

Sentri's Unique Advantages

🔓
Open source + self-hostable
No competitor is open-source. Sentri can be deployed on-prem for regulated industries (banking, healthcare, govt). This is a genuine moat — lean into it.
🦙
Private LLM via Ollama
No test code or page content leaves the org's network. Zero-data-retention compliance out of the box. No other QA platform offers this.
👁
Human-in-the-loop review
Every generated test requires human approval. Competitors auto-deploy AI tests. Sentri's model builds trust with QA teams who are skeptical of fully autonomous tools.
🌐
Multi-provider LLM flexibility
Claude, GPT-4o, Gemini, or local Ollama — swap with one setting. Competitors are locked to a single provider. Sentri won't be stranded by provider pricing changes.

Sandbox generated code in worker_threads

Replace localStorage.getItem('token') with cookie-based auth across the SPA. Add CSRF double-submit cookie.

Add MAX_RETRIES=3 and MAX_BACKOFF_MS=30000 constants to journeyGenerator.js. Emit an error event and abort the run gracefully on exhaustion.
